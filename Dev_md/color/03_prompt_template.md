```

## 5가지 테마 색상:
- Color-1 (Red/Pink): #ec1839 (RGB: 236, 24, 57)
- Color-2 (Orange): #fa5b0f (RGB: 250, 91, 15)
- Color-3 (Green): #37b182 (RGB: 55, 177, 130)
- Color-4 (Blue): #1854b4 (RGB: 24, 84, 180)
- Color-5 (Magenta): #f021b2 (RGB: 240, 33, 178)

## 라이트/다크 모드 색상:
라이트 모드:
- 메인 배경: #f2f2fc
- 카드 배경: #fdf9ff
- 보조 배경: #e8dfec
- 메인 텍스트: #302e4d
- 보조 텍스트: #504e70

다크 모드:
- 메인 배경: #171717 (너무 어둡지 않은 검정)
- 카드 배경: #242424 (약간 밝은 회색)
- 호버 상태: #2e2e2e
- 메인 텍스트: #f2f2f2 (부드러운 흰색)
- 보조 텍스트: #b3b3b3

## 구현 요구사항:
1. React Context로 테마 관리
2. localStorage에 사용자 선택 저장
3. 5가지 색상 중 선택 가능
4. 다크/라이트 모드 토글
5. 모든 버튼에 테마 색상 적용
6. 호버 효과에 rgba 투명도 활용
7. WCAG 2.1 접근성 기준 충족 (색상 대비 4.5:1 이상)

다크모드에서 폰트가 안보이거나 색약자에게 불편한 색상은 모두 수정해줘.
```

### 📋 React/TypeScript 프로젝트용 상세 프롬프트

```
React TypeScript 프로젝트에 Web_skill 컬러 시스템을 적용해줘:

## 필요한 파일들:

1. **ThemeContext.tsx** 생성
- ThemeColor 타입: 'color-1' | 'color-2' | 'color-3' | 'color-4' | 'color-5'
- ThemeMode 타입: 'light' | 'dark'
- localStorage 저장/불러오기
- 마이그레이션 로직 (기존 테마 데이터 호환)

2. **ThemeToggle.tsx** 컴포넌트
- 5가지 색상 선택 UI
- 다크/라이트 모드 토글
- 현재 선택 표시

3. **themed-button.tsx** 컴포넌트
- 테마 색상 자동 적용
- 호버 효과 (scale & color transition)
- variant: default, outline, link 지원

4. **CSS/Tailwind 설정**
```css
:root {
  --bg-black-900: #f2f2fc;
  --bg-black-100: #fdf9ff;
  --bg-black-50: #e8dfec;
  --text-black-900: #302e4d;
  --text-black-700: #504e70;
  --skin-color: #ec1839;
}

body.dark {
  --bg-black-900: #171717;
  --bg-black-100: #242424;
  --bg-black-50: #393939;
  --text-black-900: #f2f2f2;
  --text-black-700: #b3b3b3;
}
```

## 색상 스키마 객체:
```typescript
const colorSchemes = {
  'color-1': { hex: '#ec1839', rgb: '236, 24, 57', name: 'Red/Pink' },

  'color-3': { hex: '#37b182', rgb: '55, 177, 130', name: 'Green' },
  'color-4': { hex: '#1854b4', rgb: '24, 84, 180', name: 'Blue' },
  'color-5': { hex: '#f021b2', rgb: '240, 33, 178', name: 'Magenta' }
};
```

다크모드에서 모든 텍스트가 잘 보이도록 대비를 조정해줘.
```

### 📋 기존 프로젝트 마이그레이션 프롬프트

```
기존 프로젝트의 컬러 시스템을 Web_skill 컬러 시스템으로 교체해줘:

## 교체 대상:
- 모든 primary/accent 색상 → 테마 색상 변수로
- 하드코딩된 색상 값 → CSS 변수로
- 다크모드 색상 → 개선된 다크모드 팔레트로

## 마이그레이션 체크리스트:
1. 기존 localStorage 테마 데이터 마이그레이션
2. 기존 컴포넌트의 색상 클래스 교체
3. 버튼 컴포넌트를 ThemedButton으로 교체
4. 네비게이션에 테마 색상 적용
5. 다크모드에서 텍스트 가독성 확인

## 주의사항:
- 기존 테마가 'ocean', 'tech' 등인 경우 → 'color-1~5'로 매핑
- ErrorBoundary 충돌 방지를 위한 마이그레이션 로직 추가
- 다크모드 배경을 #171717로 (너무 어둡지 않게)
- 모든 텍스트 대비 WCAG 기준 충족
```

## ⚡ 문제 해결

일반적인 문제와 해결 프롬프트:

### 다크모드 텍스트가 안 보일 때:
```
다크모드에서 텍스트 가독성 문제 해결:
- 모든 텍스트 색상을 #f2f2f2 이상으로 조정
- 비활성 텍스트는 #808080 이상
- 링크 색상은 테마 색상의 밝은 버전 사용
```

### ErrorBoundary 오류 발생 시:
```
localStorage 마이그레이션 로직 추가:
- 기존 테마 데이터 체크
- 유효하지 않은 값은 기본값으로 대체
- try-catch로 안전하게 처리
```

### 버튼 호버가 작동하지 않을 때:
```
버튼 호버 효과 수정:
- onMouseEnter/onMouseLeave 이벤트 사용
- style 속성으로 직접 제어
- transform: scale(1.02) 추가
```
